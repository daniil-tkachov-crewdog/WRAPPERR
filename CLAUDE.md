# WRAPPERR — Claude Code Rules

## Git

- Always push directly to `main`. Never create feature branches unless the user explicitly asks.
- Always ask for user confirmation before pushing to `main`.

## Code Changes

- Always ask for confirmation before making any code changes.
- Every functionally(!) separate part of the code in each file should have a commented explainer that shortly mentions what this exact part of the code does, and what kind of issues or important things might be connected with this exact part of the code.
